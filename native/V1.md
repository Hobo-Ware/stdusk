# stdusk 1.0 - definition, gaps, release plan

Written 2026-07-19 against v0.2.4 (HEAD `8e02924b`), after re-verifying every
[PARITY.md](./PARITY.md) row against `native/src/` and Tabby's source. This file supersedes
PARITY.md's old "suggested next milestones" section as the roadmap to 1.0.

## (a) What "1.0" means

**stdusk 1.0 = a daily-driver replacement for Tabby for local-shell users on macOS.**

The bar, enumerated - 1.0 ships when ALL of these hold:

1. **Local-shell parity or better.** Everything a local-shell Tabby user touches daily works:
   tabs (incl. power ops), splits (nav/resize/maximize), scrollback + search, Tabby-exact
   copy/paste semantics, clickable links, themes (191 schemes + follow-OS light/dark),
   named profiles, settings GUI, session restore, quake window.
2. **North stars polished.** Progress-on-tabs, quake drop-down, ambient AI-CLI badges - the
   three reasons stdusk exists - are correct and verified, not just present.
3. **No dead ends.** A tab never silently freezes (shell exit is handled); the user's prompt
   renders correctly (their font, incl. Nerd Fonts; wide chars don't overlap); the tab title
   reflects what the shell says (OSC 0/2).
4. **Installable without incantations.** Signed + notarized `.app` via the brew cask - no
   quarantine-stripping postflight.
5. **Honest quality.** The LEDGER's "pending human verify" items (M5-M10 interactions, live
   CLI-badge detection) pass the TESTING.md checklist before the tag.

Anti-goal framing: 1.0 is NOT "every Tabby checkbox". Tabby's cosmetic window-chrome long
tail (tab-bar location, frame modes, vibrancy) and its remote-connectivity stack are out
(see §d).

## (c) Release plan

Each release ships green (fmt/clippy/tests) with LEDGER + PARITY updated.

| Release | Theme | Contents |
|---------|-------|----------|
| **0.3.0** | "No dead ends" | P0-1 shell-exit behavior, P0-3 OSC 0/2 title (+ notify-on-activity rider - same reader-thread plumbing) |
| **0.3.1** | "Your font" | P0-2 font family / Nerd Font, line padding (+ bold weight if it falls out cheap) |
| **0.3.2** | "Render right" | P0-4 wide-char correctness, minimum contrast ratio, all-match search highlight |
| **0.4.0** | "Input & scroll parity" | alt+scroll, line-step scroll hotkeys, right-click modes, focus-follows-mouse, scrollback wipe, tab leftovers (toggle-last / pin / 10-20) |
| **0.4.1** | "Panes & tabs polish" | broadcast input, aggregated tab progress/title, current-process in menu |
| **0.5.0** | "Make it yours" | profiles editor in settings GUI, `[hotkeys]` config remapping |
| **1.0.0** | "Trusted daily driver" | P0-5 signing + notarization (drop cask postflight), P0-6 TESTING.md verify pass, README/docs refresh |

Gate for 1.0.0: all P0 done; the batches above land every P1 on the way.

## (d) Non-goals for v1 (explicit)

- **Remote connectivity**: SSH client/profiles/SFTP, serial, telnet - run `ssh` in the shell.
- **Plugin system / marketplace** - a thin Rust hook API remains a FUTURE idea (PLAN §9).
- **SaaS config sync** - git-based settings sync already shipped (`sync.rs`) and is the answer.
- **Vault / encrypted secrets, zmodem, auto-sudo/UAC, analytics, custom CSS, welcome tab.**
- **Auto-update** - brew owns updates.
- **Sixel / inline images** - alacritty grid has no image model; large effort, post-v1.
- **True OpenType ligature shaping + color emoji** - egui rasterizes monochrome outlines and
  doesn't shape; symbol-substitution ligatures + monochrome emoji are the documented v1 limit.
- **Multi-window** - stdusk is a single quake window by design.
- **Non-macOS platforms** - quake/tray/shell-integration are macOS-specific today.
- **Keybinding GUI editor** (config-file remapping only) and the cosmetic window-chrome long
  tail (tab-bar location, frame modes, vibrancy, dock edges) - P2.
